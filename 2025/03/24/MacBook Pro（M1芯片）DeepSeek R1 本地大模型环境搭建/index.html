使用docker部署open-webui

地址：https://github.com/open-webui/open-webui

这个 Docker 命令启动了一个名为 open-webui 的容器，该容器在后台运行，将宿主机的 3000 端口映射到容器的 8080 端口，允许容器通过 host.docker.internal 访问宿主机，使用 open-webui 数据卷挂载到容器内的 /app/backend/data 目录，并且设置容器在任何情况下停止后都会自动重启。

-d

即 --detach 的缩写，其功能是让容器在后台以守护进程的方式运行。使用该参数后，容器启动后不会将日志输出到当前终端，而是返回容器的 ID。

-p 3000:8080

也就是 --publish 的缩写，它用于设置容器端口和宿主机端口之间的映射关系。这里将宿主机的 3000 端口映射到容器的 8080 端口，意味着通过访问宿主机的 3000 端口，就能够访问容器内监听 8080 端口的服务。

--add-host=host.docker.internal:host-gateway

该参数的作用是在容器的 /etc/hosts 文件中添加一条主机名解析记录。host.docker.internal 被解析为 host-gateway，这样容器就可以通过 host.docker.internal 这个域名来访问宿主机。host-gateway 是 Docker 提供的一个特殊值，代表宿主机的 IP 地址。

-v open-webui:/app/backend/data

即 --volume 的缩写，用于在容器和宿主机之间创建数据卷挂载。open-webui 是一个 Docker 数据卷的名称，/app/backend/data 是容器内的目标路径。这意味着将名为 open-webui 的数据卷挂载到容器的 /app/backend/data 目录下，从而实现容器和宿主机之间的数据共享与持久化。

--name open-webui

此参数为新创建的容器指定一个名称，这里将容器命名为 open-webui。之后可以使用这个名称来对容器进行管理，例如停止、启动、删除等操作。

--restart always

该参数设置了容器的重启策略为 always，也就是无论容器因为何种原因停止，Docker 都会自动尝试重新启动该容器，保证容器的持续运行。

ghcr.io/open-webui/open-webui:main

这是要使用的 Docker 镜像的完整名称，其中：

  • ghcr.io 是镜像仓库的地址，这里指的是 GitHub Container Registry（GitHub 容器注册表）。
  • open-webui/open-webui 是镜像的名称。
  • main 是镜像的标签，通常用于标识镜像的版本。
